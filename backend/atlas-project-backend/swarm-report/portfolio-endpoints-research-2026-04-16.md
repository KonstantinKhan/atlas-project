# Research: Портфельные эндпоинты

**Дата**: 2026-04-16

## Контекст запроса
Изучение REST API для управления портфелями проектов: структура, назначение и связи эндпоинтов.

## Исследованные файлы
- `routes/Portfolios.kt` — маршруты управления портфелями
- `routes/Projects.kt` — маршруты управления проектами в портфеле
- `PortfolioService` — бизнес-логика портфелей
- `ProjectService` — бизнес-логика проектов
- `CrossProjectLoadAggregator` — агрегация нагрузки ресурсов
- Доменные модели: `Portfolio`, `Project`, `PortfolioProject`
- DTO в модуле `transport` (команды, ответы)

## Ответ

Система портфелей включает 11 эндпоинтов, разделённых на три группы: **управление портфелями** (CRUD), **управление проектами в портфеле** (добавление, удаление, приоритет, переупорядочение) и **аналитика нагрузки ресурсов** (агрегированная нагрузка по портфелю). Архитектура разделяет ответственность между `PortfolioService` и `ProjectService` для операций над проектами, с поддержкой приоритизации и кросс-проектного анализа через `CrossProjectLoadAggregator`.
